mypeople

One CLI. Drive many teams of TUI agents in tmux. That's it.

The picture

you ──► mp send mypeople:eng "do the thing" │ ▼ ┌────────────────────┐ │ inbox/eng.jsonl │ durable mailbox └─────────┬──────────┘ │ drained when agent is idle ▼ ┌──────────── tmux: mc-mypeople ────────────┐ │ tab: eng → any TUI (claude/codex) │ │ tab: qa → any TUI │ │ tab: review → any TUI │ └──────────────────┬───────────────────────┘ │ agent emits lifecycle events ▼ ┌────────────────────┐ │ events/eng.jsonl │ you tail this └────────────────────┘

The CLI

mp launch mypeople
create the team (tmux session + tabs + agents)
mp send mypeople:eng "…"
queue a message to one agent
mp peek mypeople:eng
see the agent's terminal right now
mp tail mypeople
stream lifecycle events across the team
mp status mypeople
who's idle / working / dead
mp stop mypeople
graceful shutdown + cleanup

What an agent is

Input A TUI in a tmux pane that we can type into with tmux send-keys.
Output Appends lifecycle events (started, turn-ended, exited) to a JSONL file.

Claude Code already does this via a hooks plugin. For any other CLI we ship a thin shim. The runtime doesn't care which TUI runs in the pane.

v0 sketch — single machine, single boss, N teams.
