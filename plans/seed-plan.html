mypeople — SEED plan

5 seeds. Each pastable into a fresh container with claude only. Each ## Verify proves capabilities from the pane (not from agent self-report). Don't move to N+1 until N's Verify passes from a clean container.

1

Foundation

Runtime alive. No agents yet. All custom code inlined in the seed as heredocs — no git clone.
Adds
apt-install tmux python3 jq; write inline queue-server.py + queue-client.py + mp CLI from heredocs; generate queue.env with secret; launch both daemons
Verify
curl 127.0.0.1:9900/health → 200; mp status → "No active agents"; both daemons in ps
2

Core agent loop

Single-agent, local. The minimum useful thing.
Caps
spawn backend-claude send peek kill status
Verify
spawn w1; send "echo MARK"; capture-pane contains MARK; status shows w1 idle; kill; status shows no w1
3

Lifecycle notifications

Agent → Boss handoff via the per-spawn plugin.
Caps
boss-tab stop-hook status-files
Verify
spawn Boss + w1 (--boss mc-test:Boss); send w1 "reply PONG"; status/mc-test/w1.json summary = PONG; Boss pane contains [AGENT NOTIFICATION] w1 … PONG
4

Observability

The human view — dashboard and browser attach.
Caps
hud browser-attach
Verify
curl /dashboard 200; /agents API lists spawned agent; curl :7681 200
5

Cross-host via Tailscale

Goes sideways. Two containers, one Tailnet.
Caps
cross-host
Verify
two containers joined to same Tailnet, queue-server on A; spawn on B; tab exists on B (not A); message sent from A lands in B's pane; Stop notification routes back to A's Boss

Why this order

  • 1 before 2 — can't spawn agents without the queue running.
  • 2 before 3 — boss-tab is a special spawn flag; spawn must work first. Notifications need agents to notify.
  • 3 before 4 — HUD reads data that stop-hook + status-files produce; testing dashboard with zero agents is meaningless.
  • 5 last — everything proven single-host first, then scales sideways. Adds Tailscale auth as a new Input.

Anti-goals (every SEED MUST NOT)

  • Use OS package installs the seed didn't declare (beloved-engineer rule)
  • Self-report DONE without the Verify script exiting 0
  • Depend on artifacts from the host — everything inside the container
  • Skip the Interview (Step 0 collects inputs before any state change)

Self-contained constraint

  • No git clone of mypeople code. Every SEED inlines the source it ships (queue-server.py, queue-client.py, mp CLI, plugin files) as heredocs. Pasting the seed into claude is the install.
  • OS package installs (e.g. apt-get install tmux) are still declared in ## Inputs and done in Steps.
  • For SEED 5: a Tailscale auth key, collected in that SEED's Inputs.
  • Each SEED N is a superset write — it can rewrite files from N-1. ## Cleanup brings the host back to N=0 so Verify re-runs cleanly.
